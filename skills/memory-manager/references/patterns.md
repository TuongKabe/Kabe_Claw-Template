# Memory Patterns

## Daily Notes Template

```markdown
# YYYY-MM-DD

## Session Notes

- 

## Decisions

- 

## To Remember

- 

## Follow-ups

- 
```

## MEMORY.md Structure

```markdown
# Long-Term Memory

*Curated knowledge worth keeping permanently*

## Topic Areas

### Topic Name
- Key facts
- Decisions made
- References

## Last Updated

YYYY-MM-DD
```

## Commit Message Templates

### Daily Update
```
Update memory - 2026-02-03
```

### Specific Update
```
Add Zotero workflow to memory
Update crawling patterns
```

### Maintenance
```
Memory maintenance - review and distill
```

## Workflow Patterns

### Session Start Pattern
1. Read required files
2. Check/create daily note
3. Continue from where left off

### Learning Pattern
1. Capture in daily notes
2. Evaluate importance
3. Distill to MEMORY.md if significant
4. Commit

### Cleanup Pattern
1. Get recent daily notes (7 days)
2. Extract decisions and follow-ups
3. Add to MEMORY.md
4. Commit changes

## File Naming Rules

| Type | Naming | Example |
|------|--------|---------|
| Long-term memory | lowercase | `memory.md` |
| Daily notes | YYYY-MM-DD.md | `2026-02-03.md` |
| Memory directory | lowercase | `memory/` |

## Anti-Patterns to Avoid

1. ❌ "Mental notes" - always write to files
2. ❌ Skipping commits - always commit after updates
3. ❌ Uppercase filenames - always use lowercase
4. ❌ Mixing raw and curated - daily = raw, MEMORY.md = curated
